# アドレス管理表 データ追加手順

## 1. 編集するファイル
- `index.html`

## 2. 編集する場所
- `index.html` 内の以下ブロックを編集します。
- `<script id="ipData" type="application/json"> ... </script>`

## 3. 追加できるデータ
- 機器情報: `hosts`
- サブネット情報: `networks`（VLAN情報を含む）

## 4. 機器（hosts）の追加方法

### 4-1. JSON直接編集（永続化）

機器1台ごとに `hosts` 配列へ1オブジェクト追加します。

追加例:

```json
{
  "ip": "192.168.30.20",
  "name": "NAS",
  "role": "storage",
  "network_id": "home-lan",
  "assign": "static",
  "note": "バックアップ用"
}
```

フィールド一覧:

| フィールド | 説明 |
|---|---|
| `ip` | IPv4アドレス（同一JSON内で重複不可） |
| `name` | デバイス名 |
| `role` | 役割・機種名 |
| `network_id` | `networks` の `id` と一致させる |
| `assign` | `"static"` または `"dhcp"`（小文字統一） |
| `note` | 任意メモ |

> **注意:** `assign` の値はバッジ表示に影響します。`"static"` / `"dhcp"` の小文字で統一してください。

### 4-2. UI経由での操作（一時的・ページリロードで消える）

画面上で直接追加・編集・削除できます。

| 操作 | 手順 |
|---|---|
| 追加 | `+ Add Host` ボタン → フォーム入力 → 保存 |
| 編集 | 対象行の `編集` ボタン → フォーム修正 → 保存 |
| 削除 | 対象行の `削除` ボタン → 確認ダイアログで「削除する」 |

> **重要:** UI操作はメモリ上の変更です。ブラウザを更新すると `index.html` の JSON 初期値に戻ります。
> 恒久的に変更するには、操作後に内容を確認して JSON を手動で編集してください。

## 5. サブネット（networks）とVLANの追加方法

サブネット1つごとに `networks` 配列へ1オブジェクト追加します。

- サブネット一覧では以下を表示します。
  - サブネットアドレス: `cidr`
  - VLAN有無/番号: VLAN番号キーの有無で判定

VLAN番号として認識されるキー:
- `vlan_id`
- `vlanId`
- `vlan_number`
- `vlanNumber`
- `vlan.number`
- `vlan.id`

追加例（推奨は `vlan_id`）:

```json
{
  "id": "office-lan",
  "cidr": "192.168.40.0/24",
  "vlan_id": 40,
  "gateway": "192.168.40.1",
  "dhcp": {
    "enabled": true,
    "range": {
      "start": "192.168.40.100",
      "end": "192.168.40.199"
    }
  }
}
```

VLANなしの例:

```json
{
  "id": "guest-lan",
  "cidr": "192.168.50.0/24",
  "gateway": "192.168.50.1"
}
```

> ネットワークを追加すると、フィルタチップとホスト追加・編集モーダルの「ネットワークID」セレクトに自動反映されます。
> ネットワークの追加・削除はUIからは行えません。JSON直接編集のみです。

## 6. 反映イメージ

```html
<script id="ipData" type="application/json">
{
  "networks": [
    {
      "id": "home-lan",
      "cidr": "192.168.30.0/24",
      "vlan_id": 30,
      "gateway": "192.168.30.1"
    },
    {
      "id": "office-lan",
      "cidr": "192.168.40.0/24",
      "vlan_id": 40,
      "gateway": "192.168.40.1"
    }
  ],
  "hosts": [
    {
      "ip": "192.168.30.10",
      "name": "Core-Router",
      "role": "wifi-router",
      "network_id": "home-lan",
      "assign": "static",
      "note": "固定IP"
    },
    {
      "ip": "192.168.30.20",
      "name": "NAS",
      "role": "storage",
      "network_id": "home-lan",
      "assign": "static",
      "note": "バックアップ用"
    }
  ]
}
</script>
```

## 7. JSON編集時の注意
- オブジェクト同士の区切りに `,` が必要です。
- 最後の要素の末尾には `,` を付けません。
- 文字列は必ず `"` で囲みます。

## 8. 保存後の反映
- `index.html` を再読み込みすると、サブネット一覧・hosts一覧・検索結果に反映されます。
- UI経由のCRUD操作（追加・編集・削除）はページリロードで失われます。

## 9. 検索・フィルタ・ソートの仕様

| 機能 | 操作 |
|---|---|
| キーワード検索 | 検索ボックスに入力（IP / 名称 / 役割 / ネットワークID / 割当 / メモ 全列対象） |
| ネットワーク絞り込み | フィルタチップをクリック |
| ソート | テーブルヘッダ（IP / 名称 / 役割 / ネットワークID / 割当）をクリック（再クリックで昇降順切替） |
| リセット | 「クリア」ボタン（検索・フィルタ・ソートすべてリセット） |

> IPソートはオクテット数値比較です（文字列順ではなくアドレス順に並びます）。

## 10. 将来拡張（機器属性やVLANアドレス）
- 現在 `hosts` を表示・検索・CRUD対象にしています。
- 将来 `devices` や `vlanAddresses` を追加する場合は、`ipData` 内に配列を追加し、`app.js` の `dataModelAdapters` に対象定義を追加します。
